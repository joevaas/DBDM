Aim

To develop an EER model using bottom-up approach (generalization) and apply mapping cardinalities.

Algorithm
Identify low-level entities (subclasses)
Group similar entities to form a higher-level entity (superclass)
Apply generalization (bottom → top)
Define attributes for superclass and subclasses
Establish relationships between entities
Assign mapping cardinalities (1:1, 1:N, M:N)
Convert into relational schema
Concept (Use in Diagram)
Generalization (Bottom-Up)
Subclasses → Car, Truck
Superclass → Vehicle
Entities & Attributes
Vehicle (vehicle_id, type)
Car (seats)
Truck (capacity)
Owner (owner_id, name)
Relationships & Cardinality
Owner → Vehicle (1 : N)
Vehicle → Car/Truck (IS-A relationship)

👉 (Draw: Car & Truck → Vehicle using triangle “ISA”)

Code (Short SQL Mapping)
-- SQL Program to Develop EER Concept using Bottom-Up Approach (Generalization)

-- Superclass Table: Person
CREATE TABLE Person (
    Person_ID INT PRIMARY KEY,
    Name VARCHAR(50),
    Address VARCHAR(100)
);

-- Subclass Table: Student
CREATE TABLE Student (
    Student_ID INT PRIMARY KEY,
    Course VARCHAR(50),
    FOREIGN KEY (Student_ID) REFERENCES Person(Person_ID)
);

-- Subclass Table: Teacher
CREATE TABLE Teacher (
    Teacher_ID INT PRIMARY KEY,
    Subject VARCHAR(50),
    FOREIGN KEY (Teacher_ID) REFERENCES Person(Person_ID)
);

-- Subclass Table: Admin
CREATE TABLE Admin (
    Admin_ID INT PRIMARY KEY,
    Department VARCHAR(50),
    FOREIGN KEY (Admin_ID) REFERENCES Person(Person_ID)
);

-- Course Table
CREATE TABLE Course (
    Course_ID INT PRIMARY KEY,
    Course_Name VARCHAR(50)
);

-- Department Table
CREATE TABLE Department (
    Dept_ID INT PRIMARY KEY,
    Dept_Name VARCHAR(50)
);

-- Relationship: Student ENROLLS Course (M:N)
CREATE TABLE Enrolls (
    Student_ID INT,
    Course_ID INT,
    PRIMARY KEY (Student_ID, Course_ID),
    FOREIGN KEY (Student_ID) REFERENCES Student(Student_ID),
    FOREIGN KEY (Course_ID) REFERENCES Course(Course_ID)
);

-- Relationship: Teacher TEACHES Course (1:N)
CREATE TABLE Teaches (
    Teacher_ID INT,
    Course_ID INT,
    PRIMARY KEY (Teacher_ID, Course_ID),
    FOREIGN KEY (Teacher_ID) REFERENCES Teacher(Teacher_ID),
    FOREIGN KEY (Course_ID) REFERENCES Course(Course_ID)
);

-- Relationship: Admin MANAGES Department (1:1)
CREATE TABLE Manages (
    Admin_ID INT PRIMARY KEY,
    Dept_ID INT UNIQUE,
    FOREIGN KEY (Admin_ID) REFERENCES Admin(Admin_ID),
    FOREIGN KEY (Dept_ID) REFERENCES Department(Dept_ID)
);
